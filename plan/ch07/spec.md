# 项目级 MCP 配置管理 Spec

## 背景

当 MCP 安装目录（embedded-mcp-toolkit）在设置页配置完成后，还需要为具体项目启用 MCP。启用方式是在项目目录内配置两个文件：

- `.claude/settings.local.json`：通过 `enabledMcpjsonServers` 字段声明启用的 server 名称
- `.mcp.json`：通过 `mcpServers.<server>.command` 声明 server 的启动命令，其路径依赖设置页 `directory-config` 中配置的 toolkit 路径

当前项目页（ProjectsPage）为占位页，尚无任何项目级配置管理能力。用户需要手动编辑这两个文件，容易写错路径或遗漏字段，且无法直观感知配置是否正常。

## 目标

- 在项目页提供「新增项目」入口，支持添加项目卡片并以路径为标识
- 卡片自动检测目标项目两个配置文件与 toolkit 路径的健康度，以状态徽章直观呈现
- 当配置缺失或异常时，通过一键「配置」按钮完成文件创建、字段补全、路径修正，且不破坏两个文件中的其他已有字段
- 复用设置页已有的 toolkit 路径配置，路径未配置或异常时提供引导

## 功能需求

- **F1（新增项目入口）**：项目页激活时，全局 Header 右侧显示「新增项目」按钮，具备鼠标悬浮高亮效果（与「新增设备」按钮风格一致）；点击后在项目列表末尾新增一张空白项目卡片
- **F2（项目卡片路径输入）**：卡片左侧为路径输入框；输入框后紧跟「打开」按钮，具备鼠标悬浮效果，点击后弹出系统目录选择对话框；选择目录后自动填充到输入框，用户也可手动输入；填充后触发配置状态检测
- **F3（配置状态检测与徽章展示）**：填充路径后，检测目标项目目录下 `.claude/settings.local.json` 与 `.mcp.json` 的存在性及关键字段，并结合 toolkit 路径与启动脚本的健康度，以三态徽章呈现：
  - **OK**：两个文件均存在、关键字段已配置、toolkit 路径与启动脚本均正常
  - **未配置**：文件不存在或关键字段缺失（此时 toolkit 路径与脚本本身正常）
  - **配置错误**：toolkit 路径不存在或启动脚本不存在
- **F4（一键配置）**：卡片提供「配置」按钮，状态为 OK 时灰色不可点击、无悬浮效果；状态非 OK 时可点击且具备悬浮效果；点击后根据缺失情况完成「创建缺失文件 / 补全缺失字段 / 修正 command 路径」，且不影响两个文件中已存在的其他字段；当项目目录本身不存在时，配置操作先创建目录再写入；配置成功后状态自动更新为 OK
- **F5（toolkit 路径引导）**：项目页加载时，若 directory-config 中的 toolkit 路径为空或不存在，显示引导卡片并提供「去设置」按钮，点击后跳转到设置页
- **F6（项目列表持久化）**：用户添加的项目路径持久化到 settings.json 中一个**独立的新键**（与 `directory-config` 平级，不嵌套进 `directory-config` 内部），值为项目路径列表；应用重启后从该键恢复列表
- **F7（重复路径警告）**：用户输入或选择的路径与已有项目路径重复时，给出警告提示
- **F8（不存在路径警告）**：用户输入的路径不存在时，给出警告提示（允许保留该路径，因为用户可能想先通过配置按钮创建目录再写入）
- **F9（项目删除）**：每张项目卡片提供「删除」按钮，具备鼠标悬浮效果；点击后仅移除该卡片记录并清理 settings.json 中对应的项目路径记录，**不删除也不修改项目目录下的 `.mcp.json` 与 `.claude/settings.local.json`**

## 非功能需求

- **N1**：复用项目既有模式——系统目录选择（plugin-dialog）、路径防抖校验、判别联合状态分流、Toast 全局单例、组件分层约定
- **N2**：Tauri 环境降级——非 Tauri 环境下文件读写静默降级，不抛错阻断 UI
- **N3**：文件读写健壮性——目标文件不存在、JSON 解析失败、字段缺失等异常情况不抛错，统一降级为"未配置"状态
- **N4**：遵循项目编码规范（文件头注释、TypeScript 语言规范）

## 不做的事

- 不自定义 MCP server 名称（固定为 `embedded-board`）
- 不自定义启动脚本名（固定为 `remote-start-mcp.bat`）
- 不管理 MCP server 的其他参数（如 args、env 等），一键配置仅处理 `command` 字段
- 不做项目卡片的编辑能力（本次范围）
- 不做批量添加或导入
- 删除项目卡片时不删除或修改项目目录下的 .mcp.json 与 .claude/settings.local.json（仅清理应用内的记录）

## 验收标准

- **AC1（对应 F1）**：切换到项目页时 Header 右侧出现「新增项目」按钮且具备悬浮高亮；点击后列表末尾出现一张空白项目卡片
- **AC2（对应 F2）**：点击卡片「打开」按钮可弹出系统目录选择对话框；选择目录后输入框自动填充；用户手动输入路径亦可
- **AC3（对应 F3）**：面对三种典型场景，徽章分别呈现 OK、未配置、配置错误三种状态（文字与颜色可区分）
- **AC4（对应 F4）**：在「未配置」状态下点击配置按钮后，缺失的文件/字段被创建/补全，command 路径被修正为正确的 toolkit 路径，且两个文件中原有的其他字段保持不变；配置后徽章自动变为 OK
- **AC5（对应 F4 边界）**：当项目目录不存在时点击配置按钮，目录被创建后再写入配置文件，徽章变为 OK
- **AC6（对应 F5）**：toolkit 路径为空或不存在时，项目页显示引导卡片；点击「去设置」后跳转到设置页
- **AC7（对应 F6）**：添加若干项目后重启应用，项目列表恢复如初
- **AC8（对应 F7）**：输入与已存在项目相同的路径时，界面出现重复路径警告
- **AC9（对应 F8）**：输入磁盘上不存在的路径时，界面出现路径不存在警告，且该路径可被保留
- **AC10（对应 N2/N3）**：非 Tauri 环境或目标文件损坏时，页面不崩溃，状态统一降级为"未配置"
- **AC11（对应 F9）**：点击卡片「删除」按钮后，卡片从列表消失，settings.json 中 project-config 的对应记录被移除；项目目录下的 `.mcp.json` 与 `.claude/settings.local.json` 内容保持不变
