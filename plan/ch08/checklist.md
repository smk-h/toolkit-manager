# 全局 MCP 配置开关 Checklist

> 每一项通过运行代码或观察行为来验证，聚焦系统行为。

## 实现完整性

- [ ] **[F1/AC1 开关入口]** 项目页激活时 Header 右侧出现「全局 MCP」拨动开关，位于「新增项目」按钮左侧；开关有开（绿）/关（灰）两种视觉态（验证：`pnpm tauri:dev` 启动，切到项目页，观察开关位置与颜色）
- [ ] **[F2/AC2 状态检测]** `~/.claude.json` 含 `mcpServers.embedded-board` 时开关显示为开（绿）；不含时显示为关（灰）（验证：手动在 `~/.claude.json` 添加/删除该字段，重启或重新进入项目页，观察开关颜色）
- [ ] **[F3/AC3 开启操作]** 关态拨向开，弹出确认对话框；点「启用」后 `~/.claude.json` 写入 `mcpServers.embedded-board.command`，值正确（`${toolkit}/remote-start-mcp.bat` 纯正斜杠），开关变绿，toast 提示成功（验证：操作后用编辑器打开 `~/.claude.json` 确认字段写入）
- [ ] **[F4/AC4 关闭操作]** 开态拨向关，弹出确认对话框；点「关闭」后 `~/.claude.json` 的 `mcpServers.embedded-board` 被移除，其他 server 保留，开关变灰，toast 提示成功（验证：操作前后对比文件内容）
- [ ] **[F5/AC5 项目联动-禁用]** 开关开态下，所有项目卡片的「配置」按钮灰色不可点、无悬浮效果（验证：开启全局后，观察每张项目卡片的配置按钮）
- [ ] **[F5/AC5 项目联动-恢复]** 开关关态下，项目卡片「配置」按钮恢复为原本按项目状态决定的可用性（验证：关闭全局后，原本未配置的项目配置按钮恢复可点）
- [ ] **[F6/AC6 toolkit 依赖]** toolkit 路径未配置或不存在时（项目页引导态），Header 不显示全局 MCP 开关（验证：清空设置页 toolkit 路径，切到项目页，观察 Header 无开关）
- [ ] **[F7/AC7 取消回弹-开启]** 关态拨向开后在确认框点「取消」，开关回弹为关态，`~/.claude.json` 内容不变（验证：取消后观察开关颜色 + 对比文件内容）
- [ ] **[F7/AC7 取消回弹-关闭]** 开态拨向关后在确认框点「取消」，开关回弹为开态，`~/.claude.json` 内容不变（验证：取消后观察开关颜色 + 对比文件内容）
- [ ] **[N1/AC8 字段保护-其他 server]** `~/.claude.json` 中预先存在的其他 server，在开关操作后保持不变（验证：手动添加一个 `mcpServers.test-server`，开关开关一次，确认 test-server 仍在）
- [ ] **[N1/AC8 字段保护-顶层字段]** `~/.claude.json` 中预先存在的其他顶层字段（如 `theme`、`history` 等），在开关操作后保持不变（验证：手动添加顶层字段，操作后对比）

## 集成

- [ ] **[编排链路]** Header 开关拨动 → App 确认框 → 文件操作 → 状态刷新 → 开关视觉更新 的完整链路打通（验证：拨动开关，确认后观察开关变色 + 文件更新 + 项目按钮联动，四者同步）
- [ ] **[状态同步]** 全局开关状态与 `~/.claude.json` 实际内容始终一致（验证：外部手动改 `~/.claude.json` 后重启应用，开关显示与文件一致）
- [ ] **[command 一致性]** 全局写入的 command 与项目级 `buildMcpCommand` 输出完全一致（验证：全局开启后查 `~/.claude.json` 的 command，与某项目 `.mcp.json` 的 command 比对，应相同）
- [ ] **[toolkit 复用]** 全局开关与项目配置共用同一份 toolkit 路径（验证：设置页改 toolkit 路径后，全局开启写入的 command 随之变化）

## 编译与测试

- [ ] **[类型检查]** `pnpm build`（tsc + vite build）无错误（验证：运行命令，观察退出码 0）
- [ ] **[lint 风格]** 代码符合 ts-lang-spec 要求（验证：人工检查文件头注释齐全、JSDoc 完整、命名规范）
- [ ] **[文件编码]** 新建文件均为 UTF-8 无 BOM、LF 换行；修改的既有文件保持原编码不变（验证：编码检测工具核对，无乱码）

## 端到端场景

- [ ] **[E2E1 开启全流程]** 从关态完整开启：确认 toolkit 正常 → 切项目页开关为关 → 拨向开 → 确认框出现 → 点启用 → 开关变绿 + toast + 项目配置按钮全禁用 + `~/.claude.json` 写入正确 command（验证：按顺序操作，每步观察预期）
- [ ] **[E2E2 关闭全流程]** 从开态完整关闭：拨向关 → 确认框出现 → 点关闭 → 开关变灰 + toast + 项目配置按钮恢复 + `~/.claude.json` 移除 embedded-board（验证：按顺序操作，每步观察预期）
- [ ] **[E2E3 双向取消]** 关态拨开 → 取消（回弹关）→ 再拨开 → 启用（变绿）→ 拨关 → 取消（回弹绿）→ 再拨关 → 关闭（变灰）（验证：连续操作，确认每次取消都正确回弹，每次确认都正确生效）
- [ ] **[E2E4 字段保护]** 在 `~/.claude.json` 预置其他 server + 顶层字段 → 开启全局 → 关闭全局 → 确认预置内容全部保留，仅 embedded-board 被添加/移除（验证：操作前后完整对比文件）
- [ ] **[E2E5 降级健壮性]** 手动破坏 `~/.claude.json`（写入非法 JSON）→ 进入项目页 → 开关显示为关态不崩溃 → 点开启 → 文件被修正为合法 JSON 且写入配置（验证：破坏文件后观察页面不崩溃，开启后文件恢复合法）
