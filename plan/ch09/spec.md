# 配置文件手动刷新 Spec

## 背景

当前应用从外部文件读取配置后,「磁盘 → UI」方向的同步存在缺口:

- **设备配置**(外部 YAML):`useDevices` 虽有 `reload()` 能力,但仅在应用内编辑/删除/新建操作后被调用,用户在外部手动改 YAML 后 UI 不会感知
- **项目级 MCP 配置**(外部 `.mcp.json` / `.claude/settings.local.json`):`ProjectCard` 的状态检测仅在路径变化时防抖触发,外部修改文件但路径不变时不会重新检测
- **全局 MCP 配置**(外部 `~/.claude.json`):仅在应用挂载时检测一次,外部修改后开关状态不会更新
- **目录配置 / 项目列表**(应用自有 Store):仅在组件挂载时读一次

用户在编辑器、版本控制工具或其他途径手动修改上述文件后,必须切换页签(触发重新挂载)或执行某项应用内操作才能看到最新状态。需要提供统一的手动刷新入口,让用户主动触发当前页签的数据重读,无需切换页签。

## 目标

- 在 Header 顶栏为设备页和项目页提供统一的刷新按钮,位置与现有操作按钮并列
- 点击刷新后,重新读取当前页签依赖的外部配置文件,UI 立即反映磁盘最新状态
- 刷新范围限定在当前激活的页签,不做全量重读、不做定时轮询
- 刷新按钮点击后提供「刷新中」的视觉反馈,让用户确认操作已响应

## 功能需求

- **F1(刷新入口-设备页)**:设备页激活时,Header 右侧显示刷新按钮,位于「新增设备」按钮左侧;按钮含图标与「刷新」文字
- **F2(刷新入口-项目页)**:项目页激活时,Header 右侧显示刷新按钮,位于「全局 MCP 开关」与「新增项目」按钮之间;按钮为纯图标(与右侧新增按钮视觉对称)
- **F3(刷新行为-设备页)**:设备页激活时点击刷新,重新读取并解析设备目录下的全部 YAML 文件,设备列表立即反映磁盘最新内容(新增/修改/删除的设备均同步)
- **F4(刷新行为-项目页-MCP 状态)**:项目页激活时点击刷新,重新检测每个项目的 MCP 配置状态(OK / 未配置 / 配置错误),状态徽章立即更新
- **F5(刷新行为-项目页-全局 MCP)**:项目页刷新时,同步重新检测全局 MCP 开关状态(`~/.claude.json` 是否含 embedded-board),开关视觉与磁盘实际一致
- **F6(视觉反馈)**:点击刷新按钮后,按钮图标立即进入旋转动画;动画持续一段固定时长后自动停止,表明刷新操作已完成
- **F7(页签限制)**:设置页和关于页激活时不显示刷新按钮(这两个页签的数据来源是应用自有 Store,无外部文件同步需求)

## 非功能需求

- **N1(复用现有机制)**:设备页刷新复用 `useDevices` 已有的 `reload()` 能力,不新增重复的读取逻辑
- **N2(刷新失败降级)**:刷新过程中文件读取失败时静默降级,不中断 UI、不抛错阻断;保持上一次成功的数据,可由用户重试
- **N3(页签切换自动清理)**:切换页签时旧页签不再响应刷新信号(ContentArea 的 key 切换机制天然保证),无需手动清理监听
- **N4(避免重复加载)**:应用首次挂载时刷新信号不触发额外加载(各页面已有挂载时的初始加载,刷新信号仅在用户主动点击后生效)
- **N5(遵循编码规范)**:遵循项目既有的文件头注释格式与 TypeScript 语言规范(ts-lang-spec)

## 不做的事

- 不做文件监听(notify / chokidar / tauri-plugin-fs-watch 等实时监听方案)
- 不做定时轮询(setInterval 周期性重读)
- 不加 Rust 后端改动(本次纯前端实现)
- 不在设备页刷新时同步全局 MCP 状态(设备页不显示全局开关,无关联)
- 不在设置页和关于页提供刷新按钮(数据来源是应用自有 Store,外部修改场景极少)
- 不提供「刷新全部页签」的全局刷新(刷新范围严格限定当前页签)
- 不显示「上次刷新时间」「文件已变更」等提示信息(YAGNI)

## 验收标准

- **AC1(对应 F1)**:切换到设备页时,Header 右侧出现含图标与「刷新」文字的刷新按钮,位于「新增设备」按钮左侧
- **AC2(对应 F2)**:切换到项目页时,Header 右侧出现纯图标刷新按钮,位于「全局 MCP 开关」与「新增项目」按钮之间
- **AC3(对应 F3)**:在设备页点击刷新按钮后,设备列表重新加载——外部新增的设备出现、外部修改的字段更新、外部删除的设备消失(验证:在编辑器中增删改设备 YAML 文件后,点击刷新,观察列表变化)
- **AC4(对应 F4)**:在项目页点击刷新按钮后,各项目卡片的 MCP 状态徽章重新检测并更新(验证:外部修改某项目的 `.mcp.json` 后,点击刷新,观察徽章从「未配置」变为「OK」或反向变化)
- **AC5(对应 F5)**:在项目页点击刷新按钮后,全局 MCP 开关状态与 `~/.claude.json` 实际内容一致(验证:外部修改 `~/.claude.json` 的 embedded-board 字段后,点击刷新,观察开关颜色变化)
- **AC6(对应 F6)**:点击刷新按钮后,图标立即开始旋转动画,持续约 0.5 秒后停止(验证:点击刷新,观察图标旋转)
- **AC7(对应 F7)**:切换到设置页或关于页时,Header 不显示刷新按钮(验证:切到设置页/关于页,观察 Header 无刷新按钮)
- **AC8(对应 N2)**:刷新过程中目标文件被删除或内容损坏时,页面不崩溃,保持上一次成功的数据(验证:删除某设备 YAML 后点击刷新,观察列表与页面状态)
- **AC9(对应 N4)**:应用首次启动挂载时不因刷新信号触发额外的重复加载(验证:启动后观察设备列表只加载一次,无闪烁或重复请求)
