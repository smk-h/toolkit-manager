# 配置文件手动刷新 Checklist

> 每一项通过运行代码或观察行为来验证,聚焦系统行为。

## 实现完整性

- [ ] **[F1/AC1 刷新入口-设备页]** 切换到设备页时,Header 右侧出现含图标与「刷新」文字的按钮,位于「新增设备」按钮左侧(验证:`pnpm tauri:dev` 启动,切到设备页,观察按钮位置与文案)
- [ ] **[F2/AC2 刷新入口-项目页]** 切换到项目页时,Header 右侧出现纯图标刷新按钮,位于「全局 MCP 开关」与「新增项目」按钮之间(验证:切到项目页,观察三个元素的左右顺序:开关 → 刷新 → 新增)
- [ ] **[F3/AC3 设备页刷新-新增]** 在编辑器中于设备目录新增一个 YAML 文件 → 点击刷新 → 新设备出现在列表中(验证:刷新前后对比列表条目数)
- [ ] **[F3/AC3 设备页刷新-修改]** 在编辑器中修改某设备 YAML 的字段(如改通道名称)→ 点击刷新 → 设备卡片/详情反映最新字段(验证:刷新后打开详情对话框,确认字段已更新)
- [ ] **[F3/AC3 设备页刷新-删除]** 在编辑器中删除某设备 YAML 文件 → 点击刷新 → 该设备从列表消失(验证:刷新前后对比列表条目数)
- [ ] **[F4/AC4 项目页刷新-状态]** 在编辑器中修改某项目的 `.mcp.json`(如删除 embedded-board 字段使配置不完整)→ 点击刷新 → 该项目卡片徽章从 OK 变为「未配置」或反向变化(验证:刷新前后对比徽章颜色/文案)
- [ ] **[F4/AC4 项目页刷新-多卡片]** 有多个项目时点击刷新 → 所有项目卡片的徽章均重新检测(验证:刷新时观察所有卡片是否都短暂显示「检测中」后更新)
- [ ] **[F5/AC5 全局 MCP 刷新]** 在编辑器中修改 `~/.claude.json`(如手动添加/删除 mcpServers.embedded-board)→ 在项目页点击刷新 → 全局 MCP 开关颜色随之变化(验证:刷新前后对比开关颜色与文件内容)
- [ ] **[F6/AC6 旋转动画]** 点击刷新按钮后,图标立即开始旋转,持续约 0.5 秒后自动停止(验证:点击刷新,观察 RefreshCw 图标的 animate-spin 动画)
- [ ] **[F7/AC7 页签限制]** 切换到设置页或关于页时,Header 不显示刷新按钮(验证:依次切到设置页、关于页,观察 Header 右侧无刷新按钮)

## 集成

- [ ] **[信号链路-设备页]** Header 刷新按钮点击 → App.handleRefresh → refreshTick 递增 → ContentArea 透传 → DevicesPage useEffect → useDevices.reload → 设备列表更新的完整链路打通(验证:点击刷新,观察列表实时变化)
- [ ] **[信号链路-项目页]** Header 刷新按钮点击 → refreshTick 递增 → ProjectsPage 透传 refreshKey → ProjectCard 状态检测 effect 重跑 → 徽章更新的链路打通(验证:点击刷新,观察徽章变化)
- [ ] **[全局 MCP 与页面刷新同步]** 一次点击同时触发页面数据刷新 + 全局 MCP 状态重读(验证:项目页点击刷新,观察项目徽章与全局开关是否有同步更新)
- [ ] **[首次挂载不重复加载]** 应用启动时设备列表只加载一次,refreshTick=0 不触发额外 reload(验证:启动后观察设备列表无闪烁、无重复加载痕迹)

## 编译与测试

- [ ] **[类型检查]** `pnpm build`(tsc + vite build)无错误,退出码 0(验证:运行命令,观察输出)
- [ ] **[lint 风格]** 代码符合 ts-lang-spec 要求(验证:人工检查文件头注释齐全且已更新 Description、JSDoc 完整、命名规范、无 any 滥用)
- [ ] **[文件编码]** 修改的既有文件保持原编码(UTF-8 无 BOM、LF)不变,无乱码(验证:编码检测工具核对)

## 端到端场景

- [ ] **[E2E1 设备页全流程]** 启动应用 → 切到设备页 → 在编辑器中同时增/改/删设备 YAML → 点击 Header 刷新 → 图标旋转 → 列表准确反映三项变更(验证:按顺序操作,逐项核对列表与文件一致)
- [ ] **[E2E2 项目页全流程]** 启动应用 → 切到项目页 → 在编辑器中修改某项目 `.mcp.json` 使其配置失效 + 修改 `~/.claude.json` 切换全局 MCP → 点击刷新 → 图标旋转 → 项目徽章更新 + 全局开关颜色更新(验证:按顺序操作,核对徽章与开关)
- [ ] **[E2E3 页签切换不串扰]** 在设备页点击刷新(refreshTick=1)→ 切到项目页(ProjectsPage 挂载,refreshTick=1 但 effect 未追加监听,仅初始加载)→ 切回设备页(DevicesPage 重新挂载,useDevices 初始加载一次)→ 确认无意外重复刷新(验证:切换页签,观察各页签仅正常加载,无额外刷新动作)
- [ ] **[E2E4 连续刷新]** 快速连续点击刷新按钮 3 次 → 图标持续旋转 → 设备/项目列表最终与磁盘一致(验证:快速点击,观察动画与数据最终一致)
- [ ] **[E2E5 降级健壮性]** 删除设备目录下全部 YAML 文件 → 点击刷新 → 设备列表清空(显示「目录下暂无设备配置文件」),页面不崩溃(验证:删除文件后刷新,观察空列表与页面状态)
