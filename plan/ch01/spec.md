## 一、 背景

当前项目的界面为 Tauri 默认模板（Welcome 页面 + Greet 示例），不属于正式产品形态。需要完全重构为工具管理类桌面应用的正式界面，为后续章节填充真实业务功能（设备管理、项目管理、应用设置、关于信息）打好骨架。

本次工作的范围**仅限界面骨架与交互框架**：所有功能页面只展示占位提示信息，不接入任何真实业务逻辑。

## 二、 目标

- 摆脱 Tauri 模板外观，建立正式应用界面骨架
- 形成"顶部标题栏 + 左侧导航 + 右侧内容"的经典桌面应用布局
- 提供四个功能入口（设备、项目、设置、关于），通过点击切换右侧内容
- 视觉风格对齐参考项目 `farion1231/cc-switch` 的现代感（毛玻璃标题栏、品牌色激活态、切换动画）
- 为后续章节填充真实功能预留清晰的扩展位置

## 三、 功能需求

### F1：顶部固定标题栏

应用顶部存在一个固定高度的标题栏，**不随主内容滚动**。

标题栏左侧显示应用名称 `toolkit-manager`，使用品牌蓝色调。

标题栏右侧为本期预留空白区，**不放置任何控件**。

标题栏与下方内容之间有清晰的视觉分隔（背景半透明、毛玻璃模糊或分割线）。

### F2：左侧导航栏

应用左侧存在一个固定宽度的导航栏，**占满除标题栏外的全部高度**。

导航栏内含**四个功能图标项**，从上到下依次为：

| 序号 | 名称 | 用途 |
| --- | --- | --- |
| 1 | 设备 | 设备管理功能入口 |
| 2 | 项目 | 项目管理功能入口 |
| 3 | 设置 | 应用设置功能入口 |
| 4 | 关于 | 应用信息入口 |

其中"设备、项目、设置"三项顶部对齐排列；"关于"项**贴底显示**，与前三个之间有明显空隙。

每个图标项支持三种视觉状态：

- **默认态**：低调灰色
- **悬停态**：背景变浅、图标变深
- **激活态**：左侧出现品牌色竖条标记，图标变为品牌色

鼠标悬停在任一图标项上约 500ms 后，在图标右侧弹出文字气泡，显示该图标对应的功能名称。

### F3：右侧内容区与页面切换

右侧区域占据除标题栏和导航栏之外的剩余空间，用于展示当前选中功能的页面内容。

**同一时刻只显示一个功能页面**，点击左侧导航任一图标后，右侧立即切换为对应页面。

页面切换时带短暂过渡动画（淡入淡出），过渡平滑、不闪烁。

**应用启动后默认显示"设备"页面**（即"设备"图标处于激活态）。

### F4：功能页面占位内容

四个功能页面本期**不实现真实业务功能**，仅展示占位提示信息。

占位内容在页面内**垂直水平居中**，包含三个元素（从上到下）：

1. 顶部一个较大尺寸的功能图标（半透明灰色调）
2. 中间一行页面标题（正常字号、正常文字色）
3. 下方一行灰色提示文案，说明功能尚在开发中

设备、项目、设置三个页面的标题和文案如下：

| 页面 | 标题 | 提示文案 |
| --- | --- | --- |
| 设备 | 设备管理 | 该功能正在开发中，敬请期待。 |
| 项目 | 项目管理 | 该功能正在开发中，敬请期待。 |
| 设置 | 应用设置 | 该功能正在开发中，敬请期待。 |

### F4.1：关于页面的项目信息展示

"关于"页面除占位内容外，**必须展示当前项目的名称和版本号**，作为该页面独有的信息区。具体要求：

- 页面整体**垂直水平居中**布局
- 顶部一个较大尺寸的功能图标（半透明灰色调）
- 中间显示**项目名称**（`toolkit-manager`），字号大于普通文案，作为页面主标题
- 项目名称下方显示**当前版本号**（当前值为 `0.1.0`，取自项目配置），字号小于项目名称
- 最下方一行灰色提示文案：`该功能正在开发中，敬请期待。`

项目名称和版本号都必须居中显示，三者（图标 / 项目名 / 版本号 / 提示文案）从上到下排列，间距合理。

版本号必须取自项目配置（不允许在界面代码中硬编码），当前取值为 `0.1.0`。

### F5：选中状态持久化

用户当前选中的功能标签需要持久化到本地存储，应用关闭后重新打开时**自动恢复上次选中的标签**。

从本地存储读取后需校验合法性：若存储的值不在四个合法标签（设备、项目、设置、关于）范围内，则回退到默认值"设备"。

### F6：键盘快捷键（可选）

按下 `Ctrl+,`（macOS 为 `Cmd+,`）时，立即跳转到"设置"标签。

此需求为可选项，实现成本低则做，否则留待后续。

## 四、 非功能需求

### N1：主题适配

完整支持亮色和暗色两种主题。所有颜色通过语义化方式表达（不硬编码具体色值），亮色暗色下均表现正常。

本期**跟随系统主题**，不提供手动切换入口。

### N2：窗口尺寸适配

应用窗口在允许范围内缩放时，布局不应错乱、内容不应溢出或产生异常滚动条。

- 标题栏始终固定在顶部、宽度撑满
- 导航栏始终固定在左侧、高度撑满
- 内容区自适应剩余空间

### N3：视觉一致性

整体配色以品牌蓝色为基调。激活态、品牌字等关键视觉元素统一使用蓝色系，亮暗模式下均有对应表现。

间距、圆角、字号等视觉参数保持一致，避免突兀。

### N4：清理 Tauri 模板残留

正式界面建立后，**必须移除所有 Tauri 默认模板的痕迹**，包括但不限于：

- 默认的 Welcome 文案与 Greet 示例表单
- 相关的示例样式文件与示例资源（如 react logo svg）
- 后端 `greet` 命令及其调用

## 五、 不做的事

明确**不在本次工作范围内**的事项：

- 不实现任何真实业务功能（设备列表、项目 CRUD、设置项配置等）
- 不接入任何后端业务接口（"关于"页面读取版本号除外）
- 不实现登录、权限、多用户、多角色
- 不实现手动主题切换入口（只跟随系统主题）
- 不实现国际化（i18n），本期文案硬编码中文
- 不实现搜索、过滤、排序、分页等列表交互
- 不实现移动端或窄屏响应式布局（按桌面固定窗口设计）
- 不修改 Tauri 后端业务逻辑（仅清理 `greet` 命令）
- 不引入除界面骨架所需之外的状态管理库、路由库、表单库等

## 六、 验收标准

### AC1：启动默认态

应用启动后，右侧默认显示"设备"页面，左侧"设备"图标处于激活态（有蓝色竖条、图标变蓝）。

### AC2：导航切换

依次点击左侧"项目 / 设置 / 关于"图标，右侧内容立即切换为对应页面，每次切换都有淡入淡出动画。

### AC3：激活态视觉

当前激活的图标项左侧出现品牌色竖条，图标变为品牌色；其他图标保持低调灰色。

### AC4：悬停反馈

鼠标悬停在任一未激活图标上时，背景变浅、图标变深；移开后恢复默认态。

### AC5：Tooltip 显示

鼠标悬停在任一图标上约 500ms 后，图标右侧弹出文字气泡显示功能名称（如"设备"、"项目"等）。

### AC6：关于贴底

"关于"图标贴导航栏底部显示，与前三个图标之间有明显空隙。

### AC7：占位内容结构

设备、项目、设置三个功能页面中央垂直水平居中显示：大图标（半透明灰）+ 标题 + 灰色提示文案，三者间距合理。

### AC8：关于页面项目信息

"关于"页面垂直水平居中显示以下内容，从上到下排列：

1. 一个较大尺寸的功能图标（半透明灰）
2. 项目名称 `toolkit-manager`（字号大于普通文案）
3. 版本号 `0.1.0`（字号小于项目名称）
4. 灰色提示文案 `该功能正在开发中，敬请期待。`

四个元素均居中对齐，间距合理。版本号取自项目配置而非硬编码。

### AC9：状态持久化

切换到任一标签后关闭应用，重新打开应用后自动恢复到上次选中的标签。

### AC10：非法值回退

手动篡改本地存储为非法值（如 `xxx`）后重启应用，应用回退到默认显示"设备"页面，不报错。

### AC11：窗口缩放

拖动应用窗口改变大小（在合理范围内），标题栏、导航栏、内容区布局保持不乱，无内容溢出或异常滚动条。

### AC12：暗色模式

切换系统主题为暗色，整个界面颜色正确切换，所有文字、图标、背景在暗色下均可读、无对比度问题。

### AC13：模板残留清理

代码库中不再包含 Welcome 文案、Greet 示例、react logo svg、`greet` 命令等 Tauri 模板残留；应用启动后 `pnpm tauri dev` 与 `pnpm tauri build` 均可成功执行。
