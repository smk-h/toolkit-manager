## 四、 端到端场景

### 场景 1：首次启动默认态

**操作：** 清除 localStorage 中 `toolkit-manager-active-tab` 键（或首次启动），运行 `pnpm tauri dev`。

**预期：**

- 应用窗口出现，无报错
- 左侧"设备"图标（Monitor）为激活态：左侧有蓝色竖条、图标为蓝色
- 右侧显示"设备"页面：中央有 Monitor 大图标（半透明灰）+ "设备管理"标题 + 灰色提示文案
- 顶部标题栏固定，不随内容滚动

### 场景 2：完整切换四页

**操作：** 依次点击左侧"项目 → 设置 → 关于"图标。

**预期：**

- 每次点击右侧内容立即变化，且有淡入淡出动画（约 0.2s）
- "项目"页显示 `FolderKanban` 图标 + "项目管理"
- "设置"页显示 `Settings` 图标 + "应用设置"
- "关于"页显示 `Info` 图标 + `toolkit-manager`（字号较大）+ `版本 0.1.0`（字号较小）+ 提示文案
- 任一时刻只有一个图标为激活态

### 场景 3：悬停反馈与 Tooltip

**操作：** 鼠标悬停在左侧某个未激活图标上。

**预期：**

- 立即反馈：背景变浅、图标变深（hover 态）
- 悬停约 500ms 后：图标右侧弹出文字气泡，显示该图标对应的功能名（如"设备"、"项目"等）
- 移开鼠标：气泡消失，图标恢复默认态

### 场景 4：状态持久化

**操作：**

1. 启动应用，点击"设置"图标
2. 完全关闭应用窗口
3. 再次 `pnpm tauri dev` 启动

**预期：** 应用启动后自动选中"设置"标签，右侧直接显示设置页面，无需用户再次点击。

### 场景 5：非法值回退

**操作：**

1. 启动应用，关闭
2. 打开浏览器 DevTools（或在 Tauri devtools 中），将 localStorage 的 `toolkit-manager-active-tab` 改为 `"invalid_tab"`
3. 刷新或重启应用

**预期：** 应用不报错，回退到默认显示"设备"页面，"设备"图标为激活态。

### 场景 6：窗口缩放

**操作：** 拖动应用窗口边缘，在 600×400 到 1200×800 之间多次缩放。

**预期：**

- 标题栏始终固定在顶部、宽度撑满
- 导航栏始终固定在左侧、高度撑满
- 内容区自适应剩余空间，占位内容保持居中
- 无内容溢出、无异常滚动条

### 场景 7：暗色模式

**操作：** 将操作系统主题切换为暗色（Windows：设置 → 个性化 → 颜色 → 深色；或 macOS：系统设置 → 外观 → 深）。

**预期：**

- 整个界面立即切换为暗色
- 标题栏、导航栏、内容区背景均变为深色
- 文字、图标在深色背景下清晰可读，无对比度问题
- 品牌蓝色（标题字、激活态竖条、激活图标）在暗色下仍鲜明

### 场景 8：模板残留清理

**操作：** 检查代码库。

**预期：**

- `src/App.css` 已删除
- `src/assets/react.svg` 已删除（如 `src/assets/` 空则目录也删除）
- `src/App.tsx` 不含 `invoke("greet")` 调用，不含 `greetMsg`、`name` 等模板变量
- 全项目搜索 `greet` 关键字，无业务相关残留（只在历史 commit 中存在）

### 场景 9：键盘快捷键（可选）

**操作：** 应用启动后，按下 `Ctrl+,`（macOS 为 `Cmd+,`）。

**预期：** 立即跳转到"设置"标签，右侧切换为设置页面。若在 input/textarea 聚焦时按下，不触发跳转（避免误触）。

### 场景 10：生产构建产物

**操作：** 运行 `pnpm tauri build --bundles nsis`（或 `--no-bundle`）。

**预期：**

- 构建过程无错误，生成 `src-tauri/target/release/toolkit-manager.exe`
- 双击运行该 exe，应用正常启动，界面与开发模式一致
- 不因生产模式 tree-shaking 导致图标丢失、动画失效等问题
