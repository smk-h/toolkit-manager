## 一、 需求概述

当前项目界面为 Tauri 默认模板（Welcome 页面 + Greet 示例），需要完全重构为工具管理类桌面应用的正式界面。参考 `farion1231/cc-switch` 项目的视觉风格，采用"左侧导航 + 右侧内容"的经典桌面应用布局。

本次需求**只实现界面骨架与交互框架**，不实现具体业务功能。点击各功能项后右侧仅展示对应的提示信息（占位内容），后续章节再逐步填充真实功能。

## 二、 设计参考

参考项目：[farion1231/cc-switch](https://github.com/farion1231/cc-switch)

借鉴的核心设计要素：

- 固定顶部标题栏（64px 高，带毛玻璃背景）
- 单视图路由（左侧点击切换右侧内容，同时只显示一个视图）
- 路由状态持久化到 `localStorage`，下次启动恢复
- shadcn/ui + Tailwind CSS 的组件视觉风格
- 完整的亮色 / 暗色主题适配

## 四、 详细需求

### 1. 顶部标题栏（Header）

**布局**：`flex justify-between`，左右两区分布。

**左侧**：应用名 `toolkit-manager`，字体 `text-lg font-semibold`，颜色采用品牌蓝色（`text-blue-500`，暗色模式 `text-blue-400`）。

**右侧**：本期预留空白区域，不放置任何控件，后续章节再扩展（如全局设置入口、更新提醒等）。

**视觉细节**：

- 固定定位（`fixed top-0`），不随内容滚动
- 半透明背景 + 毛玻璃模糊（`bg-background/80 backdrop-blur-md`）
- 底部细分割线，与下方内容区分隔

### 2. 左侧导航栏（Nav 栏）

**布局**：垂直排列的图标列。

**尺寸**：固定宽 56px，高度撑满（`h-full`）。

**视觉风格**：

- 容器背景与主内容区有轻微区分（`bg-muted/30` 或细右边框）
- 每个图标项 40×40px，垂直居中放置图标，水平居中对齐
- 图标项之间垂直间距 8px
- 顶部留出与 Header 等高（64px）的 padding，避免被 Header 遮挡

**图标项状态**：

| 状态 | 样式 |
| --- | --- |
| 默认 | 透明背景，图标颜色 `text-muted-foreground` |
| 悬停（hover） | 背景 `bg-muted`，图标变 `text-foreground` |
| 激活（active） | 左侧 3px 蓝色竖条（`border-l-2 border-blue-500`），图标变品牌蓝色（`text-blue-500`） |

**Tooltip**：鼠标悬停 500ms 后，在图标右侧弹出文字 tooltip 显示功能名称（如"设备"、"项目"等）。使用 `title` 属性或 shadcn/ui 的 `Tooltip` 组件实现。

### 3. 四个功能标签

按从上到下的顺序：

| 序号 | 名称 | 图标（lucide-react） | 标识符 |
| --- | --- | --- | --- |
| 1 | 设备 | `Monitor` | `devices` |
| 2 | 项目 | `FolderKanban` | `projects` |
| 3 | 设置 | `Settings` | `settings` |
| 4 | 关于 | `Info` | `about` |

底部对齐方式：前三个（设备、项目、设置）顶部对齐排列，"关于"项**贴底显示**（通过 `mt-auto` 推到底部）。

### 4. 右侧内容区（Main）

**布局**：占据除 Header 和 Nav 栏之外的剩余空间。

**默认视图**：应用启动后默认选中"设备"标签。

**切换动画**：使用 `framer-motion` 的 `AnimatePresence`，切换时淡入淡出（`opacity` 过渡，duration 0.2s）。

**当前期内容**：每个功能页面**仅展示占位提示信息**，不实现真实功能。占位内容结构：

```
┌──────────────────────────────────────────┐
│                                          │
│         [大图标， muted 色]              │
│                                          │
│        XX 功能（页面标题）                │
│                                          │
│   该功能正在开发中，敬请期待。            │
│   （灰色提示文案）                        │
│                                          │
└──────────────────────────────────────────┘
```

**占位内容要求**：

- 整体垂直水平居中（`flex flex-col items-center justify-center`）
- 顶部大图标 64×64px，颜色 `text-muted-foreground/50`（半透明灰）
- 页面标题 `text-xl font-medium`，正常文字色
- 提示文案 `text-sm text-muted-foreground`，与标题间距 8px
- 内边距 `px-6`

**四个页面的占位文案**：

| 页面 | 标题 | 提示文案 |
| --- | --- | --- |
| 设备 | 设备管理 | 该功能正在开发中，敬请期待。 |
| 项目 | 项目管理 | 该功能正在开发中，敬请期待。 |
| 设置 | 应用设置 | 该功能正在开发中，敬请期待。 |
| 关于 | 关于 toolkit-manager | 版本 0.1.0 · 该功能正在开发中，敬请期待。 |

"关于"页面的提示文案中需要包含当前版本号（从 `package.json` 读取，值 `0.1.0`）。

## 五、 技术要求

### 1. 技术栈

| 项 | 选型 | 说明 |
| --- | --- | --- |
| UI 框架 | React 19 | 已有依赖 |
| 样式方案 | Tailwind CSS | **需新增**，参考项目用的就是这个 |
| 组件库 | shadcn/ui | **需新增**，提供 Tooltip、Button 等基础组件 |
| 动画 | framer-motion | **需新增**，页面切换动画 |
| 图标 | lucide-react | **需新增**，提供 Monitor、FolderKanban 等 |
| 状态管理 | React useState | 本期仅一个 `activeTab` 状态，无需引入状态库 |

### 2. 状态持久化

当前激活的标签需持久化到 `localStorage`（key: `toolkit-manager-active-tab`），应用下次启动时恢复上次选中的标签。校验逻辑：从 localStorage 读取后，若值不在 `['devices', 'projects', 'settings', 'about']` 范围内，则回退到默认值 `devices`。

### 3. 主题适配

完整支持亮色 / 暗色主题：

- 所有颜色使用 Tailwind 的语义化 class（`bg-background`、`text-foreground`、`text-muted-foreground` 等）
- 暗色模式通过 `dark:` 前缀覆盖
- 本期跟随系统主题即可，不实现手动切换入口（后续章节再加）

### 4. 键盘快捷键（可选）

- `Ctrl/Cmd + ,`：跳转到"设置"标签（与 CC Switch 行为一致）
- `Esc`：无特殊行为（本期不实现返回逻辑）
