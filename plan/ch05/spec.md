# 设备编辑与删除功能 Spec

## 背景

设备管理页面目前已具备设备列表展示、详情查看（只读）和 YAML 全文复制功能。编辑和删除按钮已占位但仅弹出"功能开发中"的 toast，无实际逻辑。

## 目标

- 点击编辑按钮弹出设备配置对话框，支持修改所有通道参数并保存回原 YAML 文件
- 点击删除按钮弹出确认对话框，确认后将 YAML 文件移入 OS 系统回收站
- 保存时通过文本级字段替换保留 YAML 文件中的注释与格式

## 功能需求

- **F1 — 编辑对话框**：点击设备卡片的编辑按钮，弹出设备配置 Dialog，按 SSH / Serial / ADB 分组展示所有可编辑字段，预填当前值
- **F2 — 增量保存**：用户修改参数后点击保存，以文本级字段替换方式写回原 YAML 文件，仅更新变化字段，保留原始注释与格式
- **F3 — 删除确认**：点击删除按钮，弹出确认 Dialog，展示设备名称及"确定要将此设备移入回收站吗？"提示
- **F4 — OS 回收站**：用户确认删除后，通过 Tauri `trash` API 将 YAML 文件移入系统回收站
- **F5 — 取消保护**：编辑/删除 Dialog 取消或关闭时，不执行任何写入/删除操作
- **F6 — 列表刷新**：编辑保存或删除成功后，自动刷新设备列表反映最新状态
- **F7 — 错误反馈**：文件写入失败、文件不存在等异常情况通过 toast 向用户提示

## 非功能需求

- **N1 — 格式保留**：编辑保存必须通过文本级字段替换实现，不得重新 dump 整份 YAML，确保注释、空行、字段顺序等原样保留
- **N2 — 安全脱敏**：编辑 Dialog 中密码（SSH password、Serial loginPassword）默认以 `·` 按长度占位显示，用户可修改；保存时传真实值
- **N3 — 幂等保存**：若用户未修改任何字段即点击保存，不应触发实际文件写入
- **N4 — 错误处理**：文件不存在、无权限、回收站不可用等异常场景应有 toast 提示，不崩溃

## 不做的事

- ❌ 新建设备配置（仅编辑已有设备）
- ❌ 设备重命名（文件名保持不变）
- ❌ 批量编辑或批量删除
- ❌ 版本历史或编辑撤销
- ❌ 项目自建回收站（直接走 OS 系统回收站）
- ❌ YAML 语法校验（假设原始文件已合法）

## 验收标准

- **AC1**：点击编辑按钮打开 Dialog，每个通道分组下各字段预填当前值
- **AC2**：修改某字段后保存，YAML 文件中对应字段值更新，注释和格式保留不变
- **AC3**：未修改任何字段时点击保存，不触发文件写入
- **AC4**：编辑保存后设备列表自动刷新，卡片摘要显示新值
- **AC5**：点击删除按钮弹出确认 Dialog，显示设备名称
- **AC6**：确认删除后 YAML 文件移入 OS 回收站（可在回收站中找到）
- **AC7**：点击删除 Dialog 取消按钮，文件保留不动
- **AC8**：对不存在的文件执行编辑或删除时，toast 提示错误且不崩溃
