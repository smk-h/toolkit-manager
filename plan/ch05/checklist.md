# 设备编辑与删除功能 Checklist

> 每一项通过运行代码或观察行为来验证，聚焦系统行为。

## 实现完整性

- [ ] **T1 — Device 类型扩展**：`Device` 接口包含 `filePath: string`（验证：编译通过；`readDevices` 返回的设备对象含有 `filePath` 字段且为完整路径）
- [ ] **T1 — Tauri 权限**：`default.json` 包含 `fs:allow-write-text-file` 和 `fs:allow-trash`（验证：查看文件确认）
- [ ] **T2 — YAML 行解析器**：`parseYamlLines()` 能正确解析 `ssh.password`、`serial.loginPassword` 等多级路径（验证：单元测试覆盖多级路径、带引号值、注释行）
- [ ] **T2 — 字段值替换**：`replaceYamlFieldValue()` 保留引号风格，`undefined` 时返回空行（验证：单元测试通过）
- [ ] **T3 — updateDeviceYaml**：调用后 YAML 文件中对应字段更新，注释与格式不变（验证：编辑保存后肉眼对比原始文件，注释和空行未被破坏）
- [ ] **T3 — trashDeviceFile**：调用后文件移入 OS 回收站（验证：打开系统回收站可找到该文件）
- [ ] **T4 — DeviceEditDialog**：组件存在并可通过编辑按钮打开（验证：页面编译通过）
- [ ] **T5 — DeviceDeleteDialog**：组件存在并可通过删除按钮打开（验证：页面编译通过）

## 功能行为

### 编辑

- [ ] 点击编辑按钮打开 Dialog，SSH/Serial/ADB 三个区块分组展示，各字段预填当前值（验证：观察各输入框内容与设备详情一致）
- [ ] 密码字段以 `·` 按原长度占位显示，input type 为 password（验证：观察密码框显示 `······`，字符不可见）
- [ ] 聚焦密码输入框时占位符清空（验证：点击密码框，`·` 消失，框为空）
- [ ] 修改任一字段后点击保存，YAML 文件中对应字段值改变，注释和格式保留（验证：用文本编辑器打开 YAML 文件确认）
- [ ] 未修改任何字段时点击保存，不触发文件写入，Dialog 直接关闭（验证：断点或对比文件修改时间戳）
- [ ] 编辑保存后设备列表自动刷新，卡片摘要显示新值（验证：卡片上 SSH host/port 等摘要信息变化）
- [ ] 点击取消或 X 关闭 Dialog，不执行任何文件写入（验证：文件内容不变）

### 删除

- [ ] 点击删除按钮弹出确认 Dialog，显示设备名称与提示文案（验证：观察 Dialog 内容）
- [ ] 确认删除后文件移入 OS 回收站，设备列表刷新，该设备消失（验证：系统回收站可见该文件，页面列表不再显示该设备）
- [ ] 点击取消或 X 关闭 Dialog，文件保留不动（验证：文件仍在原目录，列表不变）

### 错误处理

- [ ] 对已不存在的文件执行编辑保存时，toast 提示错误，Dialog 不关闭（验证：手动删除文件后点保存，观察 toast）
- [ ] 对已不存在的文件执行删除确认时，toast 提示错误，Dialog 不关闭（验证：同上）

## 集成

- [ ] DeviceEditDialog 正确调用 `updateDeviceYaml()` 并传入 `device.filePath`（验证：保存成功，YAML 文件更新）
- [ ] DeviceDeleteDialog 正确调用 `trashDeviceFile()` 并传入 `device.filePath`（验证：确认删除后文件进入回收站）
- [ ] DevicesPage 的 `reload()` 在编辑/删除成功后各被调用一次（验证：列表刷新）
- [ ] `useDevices().reload()` 重新读取后，设备列表反映最新状态（验证：编辑值生效 / 删除设备消失）

## 编译与测试

- [ ] 项目编译无错误（验证：`npx tsc --noEmit` 退出码 0）
- [ ] YAML 解析工具函数单元测试通过（验证：运行测试全部通过）
- [ ] 文件编码未被破坏：新建文件 UTF-8 无 BOM、LF 换行；修改的已有文件保持原编码不变（验证：用编码检测工具核对，无乱码）

## 端到端场景

### 场景 1：编辑设备并验证持久化

1. 在设备列表页找到一台设备，点击编辑按钮
2. Dialog 打开，修改 SSH host 字段的值（例如从 `192.168.1.1` 改为 `10.0.0.1`）
3. 点击保存
4. **预期结果**：
   - Toast 提示"设备配置已保存"
   - Dialog 关闭
   - 卡片摘要 SSH 部分显示新 host
   - 打开 YAML 文件，`ssh.host` 值已更新，注释和格式保留
   - 再次点击编辑，host 输入框显示新值

### 场景 2：删除设备并验证回收站

1. 在设备列表页找到一台设备，点击删除按钮
2. 确认 Dialog 弹出，显示设备名称
3. 点击"确定删除"
4. **预期结果**：
   - Toast 提示"设备已移入回收站"
   - Dialog 关闭
   - 该设备从列表消失
   - 打开系统回收站，可找到该 YAML 文件
   - 页面设备总数减少 1

### 场景 3：取消编辑 / 删除操作

1. 点击编辑按钮，修改一个字段，点击取消
2. **预期结果**：文件内容不变，列表不变
3. 点击删除按钮，点击取消
4. **预期结果**：文件仍在原位置，列表不变
