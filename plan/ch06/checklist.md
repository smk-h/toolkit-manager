## 功能行为

### 入口与按钮可见性（F1 / AC1 / N5）

- [ ] 当前 tab 为「设备」时，全局 Header 右侧可见「新增设备」按钮（验证：观察 Header 右侧）
- [ ] 切到项目/设置/关于任一 tab，「新增设备」按钮消失（验证：切换 tab 后检查 DOM，按钮节点不存在——不仅是隐藏）
- [ ] 按钮默认为 ghost 风格（透明背景），与 DeviceCard 操作按钮一致；鼠标悬浮时变为橙色背景白字（验证：观察默认态与 hover 态颜色变化）

### 设备名输入与校验（F2 / AC2 / AC3 / N3）

- [ ] 点击「新增设备」弹出对话框，含空的设备名输入框（验证：观察对话框内容）
- [ ] 打开对话框时输入框为空、不标红（未 touched 不报错）（验证：观察初始状态）
- [ ] 输入含 `/`、`\`、`:`、`*`、`?`、`"`、`<`、`>`、`|` 任一字符时，输入框标红并显示"设备名不能包含特殊字符..."（验证：逐个字符测试）
- [ ] 输入与现有设备同名的名称时，标红并显示"设备名「xxx」已存在"（验证：输入已有设备名）
- [ ] 输入框为空（已 touched 后清空）时，标红显示"请输入设备名"（验证：输入后清空）
- [ ] 输入合法名称时，输入框无标红、「下一步」按钮可点击（验证：输入一个新名称）

### 进入配置界面（F3 / AC4 / AC5 / N2）

- [ ] 点击「下一步」，对话框切换为配置界面，标题含设备名（验证：观察标题「配置新设备 · xxx」）
- [ ] 配置界面 SSH/Serial/ADB 三区块字段预填模板 board-example.yaml 的值（验证：对比模板文件内容，如 SSH host 为 192.168.16.105）
- [ ] 密码字段以 `·` 按长度占位显示，type 为 password（验证：观察 SSH password / Serial loginPassword）
- [ ] 聚焦密码框时占位符清空，可输入新值（验证：点击密码框 → 清空 → 输入）
- [ ] 「上一步」按钮回到设备名输入阶段，已输入名称保留（验证：点上一步，名称仍在）

### 保存与持久化（F4 / AC6 / N1）

- [ ] 未修改任何字段点保存，新文件内容与模板原文一致（验证：对比文件，逐字符相同）
- [ ] 修改某字段（如 SSH host）后保存，新文件中该字段值更新，注释/空行/uboot 段保留（验证：文本编辑器打开新文件确认）
- [ ] 新文件路径为 `<devices目录>/<设备名>.yaml`（验证：确认文件生成在正确位置，文件名与输入一致）

### 反馈与刷新（F7 / AC7）

- [ ] 保存成功后对话框关闭（验证：Dialog 消失）
- [ ] 设备列表自动刷新，新设备出现在列表中（验证：列表多出一项，名称为输入值）
- [ ] toast 提示"设备「xxx」已创建"（验证：观察 toast 文案）

### 取消保护（F6 / AC8）

- [ ] 阶段一点取消/X/ESC/遮罩，不创建任何文件（验证：检查 devices 目录无新文件）
- [ ] 阶段二点取消/X/ESC/遮罩，不创建任何文件（验证：同上）
- [ ] 关闭后再次打开，对话框恢复到阶段一、输入框为空（验证：再次点新增，状态已重置）

### 错误处理（F5 / F8 / AC9 / AC10 / AC11 / N4）

- [ ] 模板 board-example.yaml 不存在时，点「下一步」toast 提示读取失败，不进入配置界面、不创建文件（验证：临时移走模板文件后测试）

- [ ] 文件写入失败（如权限不足）时，toast 提示错误，对话框不关闭（验证：模拟不可写场景）

## 端到端场景

### 场景 1：完整新增一台设备并验证持久化

1. 切到设备页，点击 Header「新增设备」按钮
2. 输入设备名 "board-test"
3. 点击「下一步」，进入配置界面
4. 修改 SSH host 为 "10.0.0.100"（其余字段不改）
5. 点击保存
6. **预期结果**：
   - Toast 提示"设备「board-test」已创建"
   - 对话框关闭
   - 设备列表出现 board-test 卡片，SSH 摘要显示 10.0.0.100
   - devices 目录下生成 board-test.yaml
   - 打开 board-test.yaml，内容与模板一致，仅 ssh.host 为 10.0.0.100，注释/uboot 段完整保留
   - 对 board-test 执行详情/编辑/删除，均正常工作

### 场景 2：设备名校验各失败分支

1. 点击「新增设备」
2. 输入 "board/test"（含非法字符 `/`）→ 标红显示非法字符提示，「下一步」禁用
3. 改为已存在的设备名（如 board-example）→ 标红显示重名提示
4. 清空输入框 → 标红显示"请输入设备名"
5. 改为全新合法名称 → 标红消失，「下一步」可点
6. **预期结果**：三种错误状态正确切换，无新文件产生

### 场景 3：模板缺失降级

1. 临时将 board-example.yaml 移出 devices 目录（或改名）
2. 点击「新增设备」，输入合法名称，点「下一步」
3. **预期结果**：
   - Toast 提示"模板文件读取失败，请检查 board-example.yaml 是否存在"
   - 对话框停留在阶段一（设备名输入），不进入配置界面
   - devices 目录无新文件产生
4. 恢复模板文件后重试，流程正常

### 场景 4：取消保护

1. 点击「新增设备」，输入名称进入配置界面，修改若干字段
2. 点击「取消」（或 X / ESC / 遮罩）
3. **预期结果**：对话框关闭，devices 目录无新文件
4. 再次点击「新增设备」
5. **预期结果**：对话框从阶段一开始，输入框为空，上次输入与修改均已清除
