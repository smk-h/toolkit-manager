# 新增设备配置功能 Spec

## 背景

设备管理页已完成列表展示、详情查看、编辑与删除（ch05）。当前缺少"新增设备"入口——用户要新增一台设备，只能手动到 `embedded-mcp-toolkit/.embedded/configs/devices/` 目录下复制 `board-example.yaml` 再改名改参数。本期提供应用内的新增能力。

模板文件 `board-example.yaml` 是用户本地 embedded-mcp-toolkit 目录下的一份自包含示例，新增设备时以它为蓝本。

## 目标

- 在顶部全局 Header 放置「新增设备」按钮，**仅在设备页可见**
- 点击后弹出对话框，用户输入设备名并校验（非法字符 + 重复名称）
- 校验通过后进入配置界面（复用编辑表单布局），预填模板默认值，用户可修改
- 保存后将"模板文本经字段替换"的结果写入 `<devices目录>/<设备名>.yaml`，列表自动刷新

## 功能需求

- **F1 — 新增按钮入口**：顶部全局 Header 右侧放置「新增设备」按钮，仅在当前激活 tab 为「设备」时渲染；其余页面（项目/设置/关于）不显示该按钮
- **F2 — 设备名输入与校验**：点击按钮弹出对话框，含设备名输入框，用户输入时实时校验，校验失败时输入框标红并显示具体原因文案：
  - 含文件名非法字符（`/ \ : * ? " < > |`）→ "设备名不能包含特殊字符：/ \ : * ? \" < > |"
  - 与现有设备重名 → "设备名「xxx」已存在"
  - 为空 → "请输入设备名"
- **F3 — 校验通过进入配置**：设备名校验通过后，在同一对话框内切换为配置界面（分 SSH / Serial / ADB 三区块，复用编辑表单的字段布局），各字段以模板 `board-example.yaml` 的解析值为预填值
- **F4 — 参数修改与保存**：用户在配置界面可修改任意字段，点击保存后，以**模板文本 + 字段级替换**的方式生成新文件内容（保留模板注释与格式），写入 `<devices目录>/<设备名>.yaml`
- **F5 — 模板缺失降级**：若模板文件 `board-example.yaml` 读取失败或解析失败，toast 提示"模板文件读取失败，请检查 board-example.yaml 是否存在"且不进入配置界面
- **F6 — 取消保护**：对话框任意阶段取消或关闭，不创建任何文件
- **F7 — 列表刷新与反馈**：保存成功后关闭对话框、刷新设备列表（新设备出现）、toast 提示"设备「xxx」已创建"
- **F8 — 错误反馈**：文件写入失败、模板读取失败等异常通过 toast 提示，不崩溃

## 非功能需求

- **N1 — 模板格式保留**：新建设备写入的 YAML 必须以**模板原文**为基础做字段级替换，保留模板的全部注释、空行、字段顺序、uboot 子段等，不得用解析对象重新 dump 整份 YAML（与 ch05 编辑保存 N1 同源要求）
- **N2 — 密码脱敏**：配置界面中 SSH password、Serial loginPassword 默认以 `·` 按长度占位显示（与编辑 Dialog 行为一致），用户可修改
- **N3 — 实时校验**：设备名输入框的校验在用户输入过程中实时反馈（onChange），而非仅在失焦或提交时
- **N4 — 目录不存在保护**：若设备目录本身不存在（dir-missing 态），新增按钮可被点击但保存时 toast 提示错误，不应崩溃或写入到不存在的路径
- **N5 — 按钮可见性控制**：「新增设备」按钮在非设备页面完全不渲染（而非隐藏占位），避免在无关页面出现无意义入口

## 不做的事

- ❌ 模板选择（本期固定以 `board-example.yaml` 为唯一模板，不支持选其它文件作模板）
- ❌ 选择设备类型/预设配置（不做"SSH 设备模板""串口设备模板"等子模板分类）
- ❌ 设备名重命名（创建后文件名固定，重命名属于后续能力）
- ❌ 创建时多文件/多设备批量新增（一次只创建一台）
- ❌ 从零空白配置创建（必须基于模板，不提供空白 YAML 起步）
- ❌ 源码内置模板字符串备份（模板仅来自本地真实文件，读取失败即降级提示）
- ❌ 创建后自动打开编辑（创建即落盘，不强制进入二次编辑）
- ❌ 模板文件被用户删除后的恢复/重建（模板缺失仅提示，不代为创建）

## 验收标准

- **AC1**：当前激活 tab 为「设备」时，全局 Header 右侧可见「新增设备」按钮；切到项目/设置/关于任一页面，该按钮消失（DOM 中不渲染）
- **AC2**：点击「新增设备」弹出对话框，含设备名输入框，初始为空、可输入
- **AC3**：设备名输入框实时校验——输入含 `/ \ : * ? " < > |` 任一字符时，输入框标红并显示非法字符提示；与现有设备重名时标红并显示重名提示；为空时提示"请输入设备名"
- **AC4**：输入合法设备名后，对话框切换为配置界面，SSH/Serial/ADB 三区块字段以模板 `board-example.yaml` 的值为预填值（可修改）
- **AC5**：配置界面中密码字段以 `·` 按长度占位，聚焦清空、可修改（与编辑行为一致）
- **AC6**：点击保存，`<devices目录>/<设备名>.yaml` 文件被创建，内容为模板文本经字段替换的结果（注释、空行、uboot 段等原样保留）
- **AC7**：保存成功后对话框关闭、设备列表自动刷新出现新设备、toast 提示"设备「xxx」已创建"
- **AC8**：保存前点击取消或 X，不创建任何文件，列表不变
- **AC9**：模板文件 `board-example.yaml` 不存在或损坏时，点击新增按钮后 toast 提示读取失败，不进入配置界面、不创建文件
- **AC10**：设备目录不存在（dir-missing 态）时，走完校验与配置后点保存，toast 提示写入失败，不崩溃、不创建文件
- **AC11**：文件写入权限失败等异常，toast 提示错误，对话框不关闭
